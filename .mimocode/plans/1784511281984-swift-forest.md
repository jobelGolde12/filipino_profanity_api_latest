# Plan: Add Base Words Endpoint (No Leetspeak Variants)

## Goal
Create a new RESTful endpoint `GET /api/profanity/base` that returns all profanity words (filipino + regional) **without** leetspeak variants. Then test it, update docs page, and README.

---

## 1. New Endpoint: `GET /api/profanity/base`

**File:** `app/api/profanity/base/route.ts` (new file)

This endpoint is identical to `/api/profanity` but skips the variant map lookup entirely. It returns only the base words with their metadata (word, language, region, severity) — no `variants` array.

**Parameters (same as `/api/profanity`):**
| Parameter | Type | Default | Description |
|-----------|------|---------|-------------|
| `type` | string | `all` | Filter: `filipino`, `regional`, or `all` |
| `word` | string | - | Search for a specific word |
| `page` | integer | `1` | Page number |
| `limit` | integer | `50` | Items per page (max: 200) |

**Response shape:**
```json
{
  "success": true,
  "type": "all",
  "count": 50,
  "source": "database",
  "pagination": {
    "page": 1,
    "limit": 50,
    "total": 310,
    "totalPages": 7,
    "hasNext": true,
    "hasPrev": false
  },
  "data": [
    {
      "word": "gago",
      "language": "filipino",
      "region": null,
      "severity": "medium"
    }
  ]
}
```

Note: No `variants` field — just the 4 core fields per word.

**Implementation approach:**
- Copy the structure from `app/api/profanity/route.ts`
- Reuse the same helper functions (fetchFromDatabase, fetchFromJson, parsePagination, paginateData)
- Remove the variant map fetching and attachment (lines 241-245 in the original)
- The `ProfanityWord` interface omits the `variants` field
- Same rate limiting, same validation, same error handling

## 5. Test the Endpoint

After implementation, test with:
```bash
# Base URL
curl http://localhost:3000/api/profanity/base

# Filter by type
curl "http://localhost:3000/api/profanity/base?type=filipino"
curl "http://localhost:3000/api/profanity/base?type=regional"

# Search
curl "http://localhost:3000/api/profanity/base?word=gago"

# Pagination
curl "http://localhost:3000/api/profanity/base?page=1&limit=10"
```

Verify:
- Response has NO `variants` field on word objects
- All 4 base fields present (word, language, region, severity)
- Pagination works correctly
- Type filtering works
- Word search works
